## 遇到的问题和解决
1. 问题：前端页面较多，登录、鉴权、请求错误提示如果分散写在各页面中，后续维护会比较混乱。  
解决：统一抽离 `request.js` 和 `src/api/` 访问层，把请求发送、返回结构处理、token 注入和错误提示收口到公共模块中。

2. 问题：后端接口在开发阶段尚未完全稳定，前端联调和页面演示容易被阻塞。  
解决：补充 `mock-api.js`，用 mock 数据模拟登录、用户资料、识别记录和社区帖子，并支持本地持久化，保证前端页面可独立演示。

3. 问题：个人中心、登录注册页和设置页在视觉风格上原先存在不统一的情况，影响整体体验。  
解决：对登录、注册、个人中心和个人设置页进行了样式统一，调整输入框、按钮、导航栏和页面布局，使界面风格保持一致。
