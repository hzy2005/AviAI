### 2. 技术选型
- 前端框架的选择及理由：

本项目前端采用原生微信小程序框架开发。主要原因是项目目标平台就是微信生态，原生方案在页面渲染、API 调用和调试链路上最直接，和微信开发者工具配合也更稳定。相比额外引入跨端框架，原生开发可以减少适配成本，让团队把精力集中在识别流程、页面交互和接口联调上，更适合当前课程项目的开发节奏和实现深度。

- 后端框架的选择及理由：

后端选择 FastAPI。FastAPI 在接口开发效率、参数校验和文档生成方面优势明显，适合当前“快速迭代 + 明确接口规范”的项目场景。它基于 Python 生态，也更便于后续对接智能识别服务，在同一技术栈下完成业务服务与模型服务协作，能降低系统集成复杂度并提升后续维护效率。

## 心得体会
这次架构设计让我更明确了一点：先把边界和规范写清楚，比后面反复改代码更省时间。尤其是接口字段、错误码和分页结构这类细节，前期统一后，前后端联调会顺很多。

另外我也更理解“核心链路优先”的意义。像识别流程看起来简单，但把上传、推理、落库、返回这几步梳理清楚后，很多问题可以在开发前发现。整体来说，这次经历让我学会在功能实现、开发效率和维护成本之间做更实际的取舍。
