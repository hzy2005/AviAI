## 遇到的问题和解决
1. 问题：小程序端既要满足课程要求的 `frontend/src/api/` 目录，又要保持项目现有请求封装统一。
解决：新增 `frontend/src/api/` 模块目录，同时继续复用 `frontend/utils/request.js` 作为统一底层请求层。
2. 问题：前端联调时缺少稳定的演示数据。
解决：补充 Mock 处理文件和示例 `db.json`，便于截图与演示。

## 心得体会
这次作业让我更清楚前端 API 层不仅是“发请求”，更重要的是把认证、错误处理和接口组织方式统一起来。把访问层整理好之后，页面逻辑会干净很多，后续联调和功能扩展也更轻松。
